# Recipe Website UI Design

## Overview
Personal recipe web app replacing Google Drive storage. Fast, filterable, visually appealing recipe browser inspired by deniszholob.com/recipes.

## Design Decisions
- **Theme:** Auto (follows system preference) with manual dark/light toggle
- **Accent color:** Green/emerald
- **Missing photo treatment:** Solid emerald gradient placeholder
- **Detail page layout:** Two-column (ingredients left, instructions right), stacks on mobile

## Pages
1. **Homepage** (`/`) — header, search/filters, recipe card grid
2. **Recipe Detail** (`/recipe/[slug]`) — full recipe with two-column layout

## Homepage Layout (top to bottom)
1. Sticky header: site title (left), dark/light toggle (right)
2. Search bar (full width, filters titles/descriptions/tags/notes)
3. Tag filter row: AND/OR/SINGLE mode toggle + clickable tag pills
4. Result count ("Showing X of Y") + sort dropdown (alpha, date, prep time)
5. Recipe card grid (1 col mobile, 2 col tablet, 3 col desktop)
6. Footer

## Recipe Card
- Top 60%: photo with title overlay on dark gradient (or emerald gradient placeholder)
- Bottom 40%: tag pills + one-line description
- Hover: subtle lift/shadow
- Matching tags highlighted with emerald when filters active

## Recipe Detail Page
- Back link, title, tags, meta row (prep/cook time, servings adjuster, source)
- Full-width image (omitted if none)
- Two columns desktop: sticky ingredients (with checkboxes) left, instructions right
- Single column mobile: ingredients then instructions
- Print button for clean print layout

## Filtering
- Search + tags combine (AND between search and tags)
- Tag modes: AND (all tags), OR (any tag), SINGLE (one at a time)
- Client-side only, instant filtering
- Empty state with "Clear all filters" action

## Components
Header, SearchBar, TagFilter, RecipeGrid, RecipeCard, ResultCount, RecipeDetail, IngredientList, InstructionList, ServingAdjuster, EmptyState, ThemeToggle

## Data
Static JSON loaded at build time. No database.
